DISCOVER + PAY

Lightning checkout

GET /api/catalogPOST /api/payments/catchPOST /api/payments/whisperGET /api/payments/{orderId}

Send a unique Idempotency-Key with every purchase request. A quote returns HTTP 402 with orderId, amount and BOLT11. Pay it, then poll the order endpoint. Credentials are returned only after server-side settlement.

CATCH purchase example
curl -i -X POST https://the402machine.com/api/payments/catch \
  -H 'Idempotency-Key: your-unique-key' \
  -H 'Content-Type: application/json' \
  --data '{"planId":"spark"}'

# Pay the returned BOLT11, then poll:
curl https://the402machine.com/api/payments/{orderId}

WHISPER accepts client-encrypted application/octet-stream plus X-Whisper-Plan. Never upload plaintext.

CATCH / INGEST

Receive any webhook verb

POST|PUT|PATCH|DELETE|GET|HEAD|OPTIONS /c/{publicId}

Requests are accepted with or without the ingest token. CATCH records the HTTP method, content type, allow-listed headers, raw body, trusted source IP, locally resolved approximate IP location and whether the ingest token authenticated the request.

Authenticated event example
curl -X PUT https://the402machine.com/c/{publicId} \
  -H 'Authorization: Bearer {ingestToken}' \
  -H 'Content-Type: application/json' \
  --data '{"event":"canary"}'

The endpoint URL is a write capability: anyone who learns it can consume quota. Supplying the ingest token adds authenticated provenance. Owner APIs remain protected.

CATCH / OWNER

Inspect and manage

GET /api/catch/{publicId}GET /api/catch/{publicId}/eventsDELETE /api/catch/{publicId}/events/{eventId}DELETE /api/catch/{publicId}

Use Authorization: Bearer {ownerToken}. Event listing supports limit, cursor, access=public|authenticated, method, contentType and q. Bodies use bodyEncoding: "base64"; each event also includes sourceIp and best-effort ipLocation.

Paginated listing example
curl 'https://the402machine.com/api/catch/{publicId}/events?limit=20&access=authenticated&method=POST' \
  -H 'Authorization: Bearer {ownerToken}'

# Continue with nextCursor:
curl 'https://the402machine.com/api/catch/{publicId}/events?limit=20&cursor={nextCursor}' \
  -H 'Authorization: Bearer {ownerToken}'

WHISPER / ENCRYPTED DELIVERY

Encrypt before upload

POST /api/payments/whisperGET /w/{publicId}

Generate a 32-byte AES-256-GCM key locally. Upload application/octet-stream as [0x01][12-byte nonce][ciphertext + tag], with X-Whisper-Plan and Idempotency-Key. Add X-Whisper-Read-Limit: 1 to burn after the first successful read, even on Standard or Long. After settlement, keep the key outside the API response.

Encrypted retrieval
curl https://the402machine.com/w/{publicId} \
  -H 'Authorization: Bearer {readToken}' \
  --output encrypted.bin

# Decrypt locally with the separately retained AES-256-GCM key.
# Each successful retrieval uses one read. The final allowed read deletes the server copy.

The service never needs the plaintext or encryption key. The browser creates the shareable link for you.
