Everything the browser checkout does is available over HTTP. Payment settlement and provisioning are confirmed by the server.

DISCOVER + PAY

Lightning checkout

GET /api/catalogPOST /api/payments/catchPOST /api/payments/whisperPOST /api/payments/pulseGET /api/payments/{orderId}

Send a unique Idempotency-Key. A quote returns HTTP 402 with the order ID, amount and BOLT11. Poll until server-side settlement has provisioned the resource.

CATCH purchase example
curl -i -X POST https://the402machine.com/api/payments/catch \
  -H 'Idempotency-Key: your-unique-key' \
  -H 'Content-Type: application/json' \
  --data '{"planId":"spark"}'

curl https://the402machine.com/api/payments/{orderId}

WHISPER accepts client-encrypted application/octet-stream with X-Whisper-Plan. Never upload plaintext.

CATCH / INGEST

Receive webhook traffic

POST|PUT|PATCH|DELETE|GET|HEAD|OPTIONS /c/{publicId}

Requests are accepted with or without the ingest token. CATCH records bounded request data, trusted sourceIp, locally resolved approximate IP location as ipLocation, and authentication provenance. It never calls a user destination.

Authenticated event example
curl -X PUT https://the402machine.com/c/{publicId} \
  -H 'Authorization: Bearer ***' \
  -H 'Content-Type: application/json' \
  --data '{"event":"canary"}'

CATCH / OWNER

Inspect and manage

GET /api/catch/{publicId}GET /api/catch/{publicId}/eventsDELETE /api/catch/{publicId}/events/{eventId}DELETE /api/catch/{publicId}

Use Authorization: Bearer ***. Listing supports cursor, access=public|authenticated, method, content type and text filters. Bodies return an explicit bodyEncoding.

Paginated listing
curl 'https://the402machine.com/api/catch/{publicId}/events?limit=20&access=authenticated'

curl 'https://the402machine.com/api/catch/{publicId}/events?cursor={nextCursor}'

WHISPER / ENCRYPTED DELIVERY

Encrypt before upload

POST /api/payments/whisperGET /w/{publicId}

Upload opaque AES-256-GCM ciphertext with X-Whisper-Plan. Use X-Whisper-Read-Limit: 1 for burn after first read or X-Whisper-Reveal-At for scheduled reveal. The key never enters the API response.

Encrypted retrieval
curl https://the402machine.com/w/{publicId} \
  -H 'Authorization: Bearer ***' \
  --output encrypted.bin

PULSE / HEARTBEAT STATUS

Ping, inspect, publish

POST /p/{publicId}GET /api/pulse/{publicId}PATCH /api/pulse/{publicId}DELETE /api/pulse/{publicId}

PULSE accepts a bounded authenticated heartbeat. It stores no request body and never calls another URL.

Heartbeat example
curl -X POST https://the402machine.com/p/{publicId} \
  -H 'Authorization: Bearer ***'
